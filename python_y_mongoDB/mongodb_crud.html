MongoDB CRUD

Resumen de operaciones de creación, lectura, actualización y borrado en MongoDB.

  • 1. Create (Crear)
  • 2. Read (Leer)
  • 3. Update (Actualizar)
  • 4. Delete (Borrar)

1. Create (Crear)

Para insertar documentos. MongoDB crea automáticamente el campo _id si no se especifica.

  • insert_one({...}): Inserta un único documento.
  • insert_many([{...}, {...}]): Inserta un array de documentos de forma eficiente.

2. Read (Leer)

Para consultar datos. El primer parámetro siempre es el filtro.

  • find(filtro): Devuelve un cursor con todos los documentos que coincidan.
  • find_one(filtro): Devuelve solo el primer documento que encuentre.

Filtros con Operadores de Comparación

Para consultas avanzadas, no usas solo valores exactos, sino operadores.

Operadores de comparación en MongoDB
Operador Descripción Ejemplo
$eq Igual a { precio: { $eq: 20 } }
$ne No igual a { estado: { $ne: "cancelado" } }
$gt / $gte Mayor que / Mayor o igual { edad: { $gt: 18 } }
$lt / $lte Menor que / Menor o igual { stock: { $lte: 5 } }
$in Está en el array { categoria: { $in: ["IT", "RRHH"] } }
$nin No está en el array { tag: { $nin: ["old"] } }

3. Update (Actualizar)

Para modificar documentos existentes. Requiere un filtro y una instrucción de actualización.

  • updateOne(filtro, update): Modifica el primer documento que coincida.
  • updateMany(filtro, update): Modifica todos los que coincidan.

Operadores clave:

  • $set: Cambia el valor de un campo (o lo crea si no existe).
  • $unset: Elimina un campo del documento.
  • $inc: Incrementa un valor numérico.

db.users.updateOne({ _id: 1 }, { $inc: { visitas: 1 } })

4. Delete (Borrar)

Para eliminar documentos. Cuidado: si pasas un filtro vacío {}, borras todo el contenido de la colección.

  • delete_one(filtro): Borra el primer documento que coincida.
  • delete_many(filtro): Borra todos los documentos que cumplan la condición.

Drop (Eliminar colección): Para eliminar una colección completa, usa db.collection.drop(). Esta operación elimina toda la colección y sus índices. Cuidado: esta acción es irreversible.

Operadores Lógicos (Combinando Filtros)

Si necesitas filtrar por varias condiciones a la vez:

  • $and: Todas las condiciones deben ser ciertas.
  • $or: Al menos una debe ser cierta.
  • $not: Invierte el resultado del filtro.

Nota técnica: En MongoDB, si pones varias comas en un filtro { campo1: "a", campo2: "b" }, se interpreta implícitamente como un $and. Solo usas el operador explícito si necesitas repetir la misma clave o para consultas muy complejas.
